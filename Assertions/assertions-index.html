2 Entry and Exit Assertions

  1. Context: Methods/ Procs/ Functions

2.1 Entry Assertions

  1. An Entry Assertion ought to be true before entering a method.
  2. This should be guaranteed by the caller.
  3. The method assumes this.
  4. It does not check it. If it does, the Entry Assertion of the method is simply "true".

2.2 Exit Assertions

  1. An Exit Assertion is guaranteed to be true just before exiting a method.
  2. This is assumed by the caller upon return. The caller does not [have to] check it. If it does, the exit assertion does not change. Caller is merely wasting CPU cycles.
